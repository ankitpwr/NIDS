# .gitignore Configuration for NIDS1 Services

## Overview

Comprehensive `.gitignore` files have been created for each service and the repository root. These prevent sensitive data, temporary files, and large artifacts from being committed to version control.

## Files Created

### 1. Root Level `.gitignore`

**Location:** [.gitignore](.gitignore)

**Contents:**

- OS files (.DS_Store, Thumbs.db)
- All `.env` files (environment variables with secrets)
- IDE/editor configuration (.vscode/, .idea/)
- Dependencies (node_modules/, venv/, venv_new/)
- Build outputs (build/, dist/)
- Logs and temporary files
- Cache directories

### 2. ML Service `.gitignore`

**Location:** [ml-service/.gitignore](ml-service/.gitignore)

**Service-specific ignores:**

- Python virtual environment: `venv/`, `venv_new/`
- Python bytecode: `__pycache__/`, `*.pyc`, `*.pyo`
- ML artifacts (large files):
  - `artifacts/` (except `training_metadata.json`)
  - `*.pkl` (preprocessor, models)
  - `*.pt` (PyTorch weights)
  - `*.npz` (NumPy arrays)
- Flask instance folder: `instance/`
- Jupyter notebooks: `.ipynb_checkpoints/`
- Test/coverage: `.pytest_cache/`, `.coverage`, `htmlcov/`

**Note:** See "ML Artifacts" section below for storing large files.

### 3. Primary Server `.gitignore`

**Location:** [primary-server/.gitignore](primary-server/.gitignore)

**Service-specific ignores:**

- Node.js: `node_modules/`, `npm-debug.log*`, `yarn-error.log*`, `pnpm-lock.yaml`
- Build: `dist/`, `build/`, `out/`, `.next/`
- Package locks: `package-lock.json`, `yarn.lock`
- Database files (future Prisma): `*.db`, `*.sqlite`, `*.sqlite3`
- Coverage: `coverage/`, `.nyc_output/`

### 4. Frontend `.gitignore`

**Location:** [frontend/.gitignore](frontend/.gitignore)

**Service-specific ignores:**

- Python bytecode: `__pycache__/`, `*.pyc`
- Streamlit config:
  - `.streamlit/secrets.toml` (API keys, user tokens)
  - `.streamlit/config.toml` (user-specific settings)
- Jupyter: `.ipynb_checkpoints/`, `.jupyter/`
- Cache: `.cache/`, `__pycache__/`

### 5. Test Server `.gitignore`

**Location:** [test-server/.gitignore](test-server/.gitignore)

**Service-specific ignores:**

- Python virtual environment: `venv/`, `venv_new/`
- Python bytecode: `__pycache__/`, `*.pyc`
- Packet captures:
  - `*.pcap`, `*.pcapng` (Scapy/Wireshark captures)
  - `pcap/`, `captures/` directories
- Agent/sniffer: `.scapy_history`
- Flask: `instance/`, `.webassets-cache`

## What SHOULD Be Committed

✅ **Commit these:**

- Source code (`.py`, `.js`, `.ts` files)
- Configuration templates: `.env.example`, `config.template.json`
- Documentation: `README.md`, `*.md` files
- Requirements files: `requirements.txt`, `package.json`
- `.gitkeep` (to preserve empty directory structure)
- License files: `LICENSE`
- CI/CD configuration: `.github/`, `.gitlab-ci.yml`
- Prisma schema: `schema.prisma`
- Small test data: `tests/fixtures/` (if < 1MB)

## What Should NOT Be Committed

❌ **Never commit these:**

- `.env` files (environment variables with secrets)
- API keys, passwords, tokens
- `node_modules/` (too large, regenerated by npm install)
- `venv*/` (too large, regenerated by pip install)
- `.pyc`, `__pycache__/` (generated from source)
- Large ML artifacts (`*.pkl`, `*.pt`, `*.npz`) > 10MB
- Local database files
- IDE settings (`.vscode/settings.json` user-specific)
- OS files (`.DS_Store`, `Thumbs.db`)

## ML Artifacts Strategy

The ML models are large files that shouldn't be in regular Git. Options:

### Option 1: Git LFS (Recommended for production)

```bash
# Install Git LFS
git lfs install

# Track large files
git lfs track "*.pkl"
git lfs track "*.pt"
git lfs track "*.npz"
git add .gitattributes

# Add artifacts
git add ml-service/artifacts/
git commit -m "Add ML artifacts with Git LFS"
```

### Option 2: Separate Storage (Current approach)

```bash
# Uncommit in ml-service/.gitignore:
artifacts/
!artifacts/training_metadata.json  # only metadata

# Instead, store artifacts in:
# - AWS S3
# - Google Cloud Storage
# - Azure Blob Storage
# - Mounted volume in Docker/K8s
```

### Option 3: Build/Download in CI/CD

```bash
# In CI/CD pipeline (GitHub Actions, GitLab CI):
- Download artifacts from storage during build
- Models loaded at runtime, not stored in repo
```

## Verification

### Check what will be tracked:

```bash
# Show all untracked files (excluding .gitignore)
git check-ignore -v $(git ls-files --others --exclude-standard)

# Show what's currently tracked
git ls-files
```

### Verify sensitive files are ignored:

```bash
# Should return nothing if .env is properly ignored
git check-ignore .env

# Should return the file if it's missing from .gitignore
git check-ignore ml-service/venv_new/
```

## Common Issues & Solutions

### Issue: .env files committed accidentally

```bash
# Remove from Git history (keep local copy)
git rm --cached .env
git commit -m "Remove .env from tracking"

# Or remove entire history
git filter-branch --tree-filter 'rm -f .env' HEAD
```

### Issue: Artifacts committed (large files)

```bash
# Remove large file from history
git filter-branch --tree-filter 'rm -f ml-service/artifacts/*.pt' HEAD

# Use BFG Repo-Cleaner for faster cleanup
bfg --delete-files "*.pt" .
```

### Issue: node_modules/ or venv/ committed

```bash
# Add to .gitignore and remove from tracking
echo "node_modules/" >> .gitignore
git rm -r --cached node_modules/
git commit -m "Remove node_modules from tracking"
```
